# Specs

This directory stores implementation specs that are ready to guide coding.

Each spec should answer:

- What exactly will be implemented now
- Which files, modules, flows, or views are involved
- What the acceptance criteria are
- What is explicitly not included in this step

Rule:

- A non-trivial item should not move into implementation without a confirmed spec
- A spec should point back to its requirement and design
- A spec should describe a scoped increment, not an entire product rewrite
- A spec should include acceptance criteria and test design
