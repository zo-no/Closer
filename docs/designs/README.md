# Designs

This directory stores design documents after a requirement is clear.

Each design should answer:

- What form the solution should take
- What alternatives were considered
- What boundaries and tradeoffs are being chosen
- What risks or open questions remain

Rule:

- A design should point back to its requirement
- A design is still not the coding plan
- A design should reduce ambiguity enough that a spec can be written cleanly
- A design should end with clear boundaries, open questions, and handoff notes for spec writing
