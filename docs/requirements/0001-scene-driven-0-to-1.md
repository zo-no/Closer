# 场景驱动的高质量认识与 0-1 推进

## Background

Closer 的第一版 MVP 不再试图覆盖完整恋爱流程，而是先聚焦在一个更前置、更真实的核心问题上：

- 用户如何进入更适合自己的认识场景
- 如何在有限时间下高质量地认识异性
- 如何把关系从陌生人推进到初步稳定互动

当前项目来自真实个人问题，而不是纯产品假设。现阶段的核心目标，是先解决创始人自己在认真恋爱场景中的真实卡点，再将这套方法论抽象为可以产品化的能力。

## Problem

当前的主要问题不是聊天能力不足，也不是完全没有机会，而是以下几类问题叠加：

1. 主动进入新场景的频率偏低
2. 现有线上渠道能聊起来，但很难遇到真正心动和愿意主动推进的人
3. 时间和精力有限，不愿意在低质量机会里持续消耗
4. 缺少一套围绕“合适场景 -> 自然认识 -> 初步推进”的方法论和执行闭环

旧 MVP 更像一个 dating ops dashboard，强调机会池、任务、地图和见后复盘，但没有解决“去哪里认识真正合适的人”这一前置问题，也没有体现创始人对浪漫感、场景感和高质量相遇的重视。

## Target User

第一版目标用户先以创始人本人为原型，后续再抽象出通用模板。

当前原型用户特征：

- 年龄：23 岁左右的年轻程序员
- 城市：北京
- 目标：认真恋爱，而不是随意扩列
- 现实约束：工作节奏快，时间有限，不适合高频低质量社交
- 偏好对象：
  - 年龄相近
  - 气质干净、阳光、偏日系风格
  - 能进行深入对话和思维共鸣
- 偏好相遇方式：
  - 场景自然
  - 具备浪漫感和故事感
  - 不喜欢纯工具化、纯效率化的冷启动方式

潜在高价值场景包括但不限于：

- 读书活动
- 徒步活动
- 户外活动
- 摄影活动
- 带有文艺气质和自然互动机会的线下场景

## Goals

第一版 MVP 的目标是帮助用户：

1. 找到更高概率出现目标对象的场景
2. 建立一套适合自己的认识和推进方法，而不是只提供零散建议
3. 降低主动进入场景和主动推进的心理阻力
4. 把“认识人”从模糊、随机、低反馈的行为，变成可执行、可复盘、可逐步优化的过程

## Scope

本期 requirement 先覆盖以下内容：

1. 场景选择问题
   - 用户适合去哪里认识更可能让自己心动的人
   - 如何筛选对自己有效、可持续进入的场景

2. 0-1 推进问题
   - 从初次认识到初步稳定互动的关键步骤是什么
   - 哪些动作和节奏能自然推进，而不显得机械

3. 方法论与产品结合
   - 这个问题不能只靠软件功能解决
   - 需要结合方法论、引导、模板、记录和教学内容

4. 数据记录基础
   - 用户当前对自己的漏斗数据认识很模糊
   - 后续可考虑把场景、接触、持续互动、推进结果等数据纳入记录

## Out of Scope

第一版 requirement 暂不重点解决：

- 如何进入长期恋爱关系
- 如何经营恋爱关系
- 复杂的匹配算法
- 广泛的聊天技巧库
- 见面之后的深度关系管理
- 多用户通用平台级系统设计

## Success Criteria

这一期 requirement 达成的标准不是已经把功能做完，而是明确下面这些问题：

1. 第一版 MVP 要解决的用户问题边界被锁定
2. 产品不再被误导成 dating dashboard 或纯聊天工具
3. “场景驱动的高质量认识”被确认是第一阶段核心方向
4. 后续 design 可以围绕“场景、行动、推进、复盘”展开，而不是泛泛做恋爱助手

## Notes

- 创始人的真实情况会作为第一份种子用户研究样本
- 后续可以把这套分析框架抽象成通用模板，用于分析不同用户的“认识人”阶段问题
- 当前一个关键开放点是：哪些线下场景最适合成为第一版 MVP 的重点支持对象
