# Requirements

This directory stores product requirements.

Use a requirement document to define the real problem before discussing solutions.

Each requirement should answer:

- What problem are we solving
- Who is affected
- Why it matters now
- What is in scope
- What is out of scope
- What success looks like

Rule:

- Do not write implementation details here
- Keep the focus on problem, scope, and success criteria
- A confirmed requirement should hand off cleanly into design
- Do not begin implementation from requirement discussion alone
